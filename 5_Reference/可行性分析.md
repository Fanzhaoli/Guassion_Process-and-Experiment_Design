### 5. Experimental Design Summary

This is a **Self-Matching Task (SMT)** paradigm. Key features:

- **Shapes**: Two shapes ("circle", "square")
- **Labels**: Two identity labels ("self", "stranger") assigned to shapes
- **Matching definition**: `(circle + self)` or `(square + stranger)` = Matching; others = NonMatching
- **Design factors (P, T, W)** systematically varied across 6 groups:
  - **P** (Practice): ranges 0-120
  - **T** (Presentation time): ranges 30-500 ms (stored as 0.03-0.5 s)
  - **W** (Response window): ranges 300-1500 ms (stored as 0.3-1.5 s)

Key data pattern: A large proportion of trials have **missing RT** (empty RT and Response fields), indicated by `Correct=0`. These are **omission trials** -- trials where the subject did not respond within the response window (W). This is a known feature of the deadline-based DDM with short presentation times.

### 6. SPE (Self-Prioritization Effect) Statistics

The SPE is computed as the mean RT difference:

```
SPE = mean(RT_self) - mean(RT_stranger)
```

A negative SPE means subjects respond **faster** to self-associated stimuli.

Based on the project's existing analysis ([SPE_Bayesian_Analysis.ipynb](file:///d:/GitHub_programe/GitHub/Guassion-Process-Experiment-Design/1_Code/Python_for_Check/ANOVA/SPE_Bayesian_Analysis.ipynb)), the data is analyzed with:

- **RT analysis**: Only correct-response trials (ACC == 1), RT in seconds
- **ACC analysis**: All trials (including omissions)
- **SPE range** (from AGENTS.md): approximately **-90 ms to +4 ms**, varying by P/T/W group

The AGENTS.md notes that the real SPE range is about **-90ms to +4ms**, which is modulated by P/T/W design factors. This means self-associated stimuli produce faster RTs by up to 90ms compared to stranger-associated stimuli, and this effect varies systematically across experimental conditions.

## 逐层分析：概念、优势与潜在问题

### 第一层：Sigmoid 参数生成 — ✅ 有理论基础，但需要校准

你使用 Sigmoid 函数将 P、T、W 映射为 DDM 的漂移率（v）和边界分离度（a），这在心理学建模中是有前例的——许多模型假设实验变量通过某种单调/饱和函数影响认知参数。你的 `compute_v_s2` 和 `compute_a_s2` 体现了以下核心假设：

- **P → v**：练习次数通过逻辑函数影响漂移率（`v_P_Function`），暗示练习效应存在饱和
- **T → v**：刺激呈现时间也通过 Sigmoid 影响漂移率，T₀=100ms 作为中心点
- **M → a**：M = T + W（总可用时间）影响边界分离度

**✅ 可取之处**：这套函数提供了一个参数化的理论假设——你可以确切地说"我们假设 P/T/W 通过如下数学形式影响认知过程"。

**⚠️ 需要注意的问题**：

1. **参数缺乏来自真实数据的校准**。`alaph1=1.5`、`alaph2=-0.4`、`k_T=0.01` 等参数的取值目前是手工设定的，而非通过拟合真实数据获得。从你的 step1 检验结果来看，v2.4 生成的 SPE ≈ -200ms（Cohen's d ≈ -1.02），而真实数据的 SPE 范围约为 -90ms 到 +4ms。**模拟数据中的自我优势效应约为真实数据的 2-5 倍**，这说明参数需要重新校准。
2. **Sigmoid 函数的心理学可解释性**值得进一步讨论。比如 `v_T = 1 / (1 + exp(-0.01*(T-100)))`，当 T 从 10ms 变到 600ms 时，这个函数的变化范围非常有限（因为 k_T=0.01 太小），实际上 T 的变化对 v 的贡献可能被压缩得过于平缓。

## 综合可行性判断与建议

### 我认为这个项目的核心研究问题是可行的：

> "能否使用 GP 模型来描述和预测 SPE 在实验设计空间 (P,T,W) 中的变化，从而指导实验设计优化？"

**答案为：是的，这个方向在方法论上是合理的，而且在计算精神病学、贝叶斯认知建模领域有大量先例。** 但目前的实现方案需要进行一些关键调整。

### 具体建议（按优先级排序）：

#### 建议1：重新定位 GP 的角色

我建议将 GP 的角色从"生成器中的混合组件"转变为"设计空间中的响应面模型"。具体来说：

**当前做法**：

```
GP 注入任意残差 → 生成模拟数据 → 做参数恢复
```

**建议做法**：

```
真实数据 → HDDM/DDM 拟合 → 获得每个条件的 (v, a, t0) 真值
         ↓
    GP 学习 P,T,W → (v, a, SPE) 的映射
         ↓
   GP 响应面 + 不确定性估计 → 实验设计优化
```

#### 建议2：用真实数据校准 Sigmoid 参数

- 使用 HDDM（Python 的 `hddm` 包）或自定义 MCMC 对 46 名被试的数据拟合 DDM
- 提取每个被试在每个条件下的 DDM 参数估计值
- 用这些"经验"DDM 参数来校准 `compute_v_s2` 和 `compute_a_s2` 中的参数（alaph1, alaph2, beta1, k_T 等）

#### 建议3：GP 直接建模 SPE 而非 DDM 参数残差

你的探索 Notebook（2D 和 3D）实际上已经展示了更有前景的路径：用 GP 直接预测 SPE 效应量作为 P、T、W 的函数。这样做的好处是：

- 直接回答核心研究问题
- 避免 DDM 参数→行为→SPE 的多层间接推断
- 可以自然地结合真实数据

#### 建议4：渐进式验证策略

| 阶段    | 内容             | 数据                            |
| ------- | ---------------- | ------------------------------- |
| Layer 1 | Sigmoid 参数校准 | 真实数据 DDM 拟合               |
| Layer 2 | GP 响应面建模    | 真实数据的 SPE 或 DDM 参数      |
| Layer 3 | 生成模型验证     | 对比模拟 vs 真实数据的 SPE 分布 |
| Layer 4 | 实验设计优化     | GP 响应面 + 采集函数            |

## 一、一个关键澄清：SPE 在 SMT 中的精确定义

从 [SPE_Bayesian_Analysis.ipynb](file:///d:/GitHub_programe/GitHub/Guassion-Process-Experiment-Design/1_Code/Python_for_Check/ANOVA/SPE_Bayesian_Analysis.ipynb#L119-L123) 可以看出，SMT 的设计逻辑是：

```
学习阶段: circle → self, square → stranger (或反向平衡)
          ↓
测试阶段:
  Matching = (circle, self) 或 (square, stranger)   → 按 'f' 键
  NonMatching = (square, self) 或 (circle, stranger) → 按 'j' 键
          ↓
SPE = RT(self, Matching) - RT(stranger, Matching)
```

**SPE 只在 Matching 条件下计算**。这是 Sui et al. (2012) 的做法，因为 Matching 条件下刺激-身份配对是"一致"的，自我优势效应的表达最纯粹。

## 二、NonMatching 条件能否用 DDM 拟合？

这是一个非常好的理论问题。答案是：**原则上可以，但需要仔细考虑模型的解释框架。**

### 2.1 技术上可以拟合

DDM 本质上是二元选择模型——只要被试在两个选项之间做决策，DDM 就可以拟合。NonMatching 试次中，被试看到 stimulus + identity 组合同样进行"按 f 还是 j"的二元决策，所以数学上 DDM 是完全适用的。

### 2.2 但 NonMatching 有独特的认知机制需要建模

这是问题的核心。NonMatching 条件涉及**反应冲突（response conflict）**：

| 条件        | 刺激-身份关系               | 认知过程                         |
| ----------- | --------------------------- | -------------------------------- |
| Matching    | circle-self → 确认 → 按 f | 一致：自动加工与任务要求一致     |
| NonMatching | square-self → 抑制 → 按 j | 冲突：自动偏向被抑制，需认知控制 |

在 DDM 框架中，反应冲突有几种建模方式：

- **漂移率降低**：冲突增加了决策难度，v 降低
- **起始点偏移**：z 偏离 a/2，反映对某个反应的偏向
- **变化性变异**：冲突增加了试次间的变异性

所以，**如果你想对 NonMatching 进行 DDM 拟合，你需要明确 NonMatching 条件下认知过程的特殊性，而不是简单套用 Matching 的参数结构。**

## 四、关于高遗漏率（NA）与 HDDM 拟合

### 4.1 遗漏试次的 DDM 处理

你提到"T+W 太短时被试反应不过来，有很多 NA 试次"。这实际上不是 DDM 的"bug"，而是 DDM 可以建模的**特征**：

在 standard DDM 框架中，一个试次如果未能在截止时间（你的 W）内达到决策边界，会被截断（truncated）。这被称为**截尾数据（censored data）**或**右删失（right-censored）**。HDDM 实际上支持对这类数据的处理——通过设置 `rt` 为截止时间并标记为 censored。

从你的原始数据看：

```
group1_1: P=0, T=30ms, W=300ms
→ 大量 NA（总试次 520，有效按键仅约 162 个/人）
```

条件 1 (P=0, T=30, W=300) 的遗漏率确实非常高（约 71%），这确实对 DDM 拟合构成挑战。但 Docker HDDM 可能可以通过 hierarchical Bayesian 结构部分缓解这个问题——因为层级模型可以"借用"其他条件的参数信息来稳定对稀疏数据的估计。

### 4.2 Docker HDDM 的适用性

你提到的 `hcp4715/hddm` Docker 镜像（基于 Wiecki et al. 2013 的 HDDM 包）是一个成熟的选择。它的层级贝叶斯结构在这种情况下有价值，因为：

- 被试内部试次少 → 层级模型共享群体分布信息
- 条件间参数可关联 → 稳定极端条件的估计
- 内置 censored data 处理

**使用建议**：在拟合时，将未反应试次标记为 `censored=True`，RT 设置为 W+T（截止时间）。特别关注条件 1-4（遗漏率高）的后验收敛情况。
